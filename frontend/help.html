Camera access works only in secure contexts such as localhost or HTTPS.

Best practice:
• Use Live Server or localhost during development.
• Do not open the file with file://.
• Check browser permission settings and allow camera access.
• If needed, refresh once after granting permission.

In this version, history is often stored in localStorage. If the browser data is cleared, the history may be removed.

Check the following:
• Use the same browser and same origin.
• Confirm that save functions are writing to localStorage.
• Verify that your history page reads the correct keys.
• Do not switch between unrelated ports or domains while testing.
